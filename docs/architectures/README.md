# sae-forge — model architectures, in n-orca

Each architecture below has been converted from its Hugging Face `config.json` into a verified [n-orca](https://github.com/jascal/n-orca) document. The same conversion runs via the `n-orca` MCP server (registered with Claude Code as `n-orca`) — call the `convert_from_hf` tool with any HF model id to regenerate.

Each entry has:

- `<name>.n.orca.md` — the verified n-orca architecture document
- `<name>.mmd` — a Mermaid flowchart of the architecture

## Not yet supported

These model families ship in the parent project's adapter list but n-orca v0.1 does not yet model their topology. See [`_unsupported/`](_unsupported/) for per-model context.

| Model | Hub id | Why |
|-------|--------|-----|
| whisper-base | [`openai/whisper-base`](https://huggingface.co/openai/whisper-base) | Encoder-decoder speech model — n-orca v0 does not yet model encoder-decoder topologies; would need a `WhisperAdapter` covering log-mel CNN stem + encoder + cross-attention decoder. |
| qwen3-moe | [`Qwen/Qwen3-30B-A3B`](https://huggingface.co/Qwen/Qwen3-30B-A3B) | Qwen3 mixture-of-experts decoder — n-orca v0 represents FFN as a dense block; a `MoeFeedForward` op (with `n_experts` and `top_k` parameters) would be the minimum required addition. |
