# architecture-adapters Specification (delta)

## ADDED Requirements

### Requirement: Nanochat-family architecture adapter

A `NanochatAdapter` SHALL be registered against the `lm_sae.train.GPT` class (the nanochat-style single-file GPT used by the `lm-sae` fixture and `karpathy/autoresearch` family).

Registration SHALL be lazy:

- If `lm_sae` is not importable, the adapter module SHALL NOT register and SHALL NOT raise. `saeforge.adapters.adapter_for(any_host)` continues to work for all other registered families.
- If `lm_sae` is importable, the adapter SHALL be registered such that `adapter_for(host)` returns a `NanochatAdapter` instance whenever `isinstance(host, lm_sae.train.GPT)`.

The adapter's `family` class attribute SHALL be the string `"nanochat"`.

### Requirement: Nanochat walk parameter inventory

`NanochatAdapter.walk(host, projector, *, attention_width="host")` SHALL emit a `dict[str, np.ndarray]` containing exactly these keys, projected through the supplied `SubspaceProjector`:

- `transformer.wte.weight` (projected via `project_embed`)
- For each layer `i` in `0..host.config.n_layer - 1`:
  - `transformer.h.{i}.attn.c_q.weight` (projected via `project_qkv_full`)
  - `transformer.h.{i}.attn.c_k.weight` (projected via `project_qkv_full`)
  - `transformer.h.{i}.attn.c_v.weight` (projected via `project_qkv_full`)
  - `transformer.h.{i}.attn.c_proj.weight` (projected via `project_attn_out`)
  - `transformer.h.{i}.mlp.c_fc.weight` (projected via `project_mlp_in`)
  - `transformer.h.{i}.mlp.c_proj.weight` (projected via `project_mlp_out`)
- For each layer `i` that has a value embedding (per `has_ve(i, n_layer)`):
  - `value_embeds.{i}.weight` (projected via `project_embed`)
  - `transformer.h.{i}.attn.ve_gate.weight` (projected via `project_residual_aligned`)
- `lm_head.weight` (projected via `project_lm_head`)
- `resid_lambdas` (1D scalar tensor, passed through unchanged)
- `x0_lambdas` (1D scalar tensor, passed through unchanged)

The walk SHALL NOT emit:

- `cos` or `sin` buffers (regenerated by the forged module at construction time)
- `ln_*` parameters (nanochat's RMSNorm is parameter-free)
- `wpe` (nanochat uses rotary only)

### Requirement: Acceptance gate for the first forge run

The integration test in `tests/test_nanochat_adapter.py::test_first_forge_run` SHALL produce a `retained_mauc` value strictly greater than 0.55 over the subset of token-identity probes with at least 50 positives in the eval window. Absolute value above this threshold is the gate; numeric tuning is out of scope for this change.
